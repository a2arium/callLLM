# Retrieval with Embeddings

Start with [Embeddings](embeddings.md) for the basic API. This guide focuses on retrieval systems: semantic search, RAG indexes, and model consistency.

## The Main Rule

Use the same exact embedding model for indexing and querying.

```ts
import { LLMCaller } from 'callllm';

const EMBEDDING_MODEL = 'text-embedding-3-small';
const caller = new LLMCaller('openai', { model: EMBEDDING_MODEL });

const documentEmbedding = await caller.embeddings({
  input: 'Refunds are processed within five business days.',
  model: EMBEDDING_MODEL
});

const queryEmbedding = await caller.embeddings({
  input: 'When will I get my money back?',
  model: EMBEDDING_MODEL
});
```

Do not mix embedding models inside one vector index. Different embedding models can have different dimensions and different vector spaces, so cosine similarity may become meaningless or impossible.

## Store Model Metadata

Persist the model and dimensions beside every stored vector:

```ts
type StoredVector = {
  id: string;
  text: string;
  embedding: number[];
  embeddingModel: string;
  dimensions: number;
};

const result = await caller.embeddings({
  input: documentText,
  model: EMBEDDING_MODEL
});

const row: StoredVector = {
  id: documentId,
  text: documentText,
  embedding: result.embeddings[0].embedding,
  embeddingModel: result.model,
  dimensions: result.embeddings[0].embedding.length
};
```

This makes migrations explicit and prevents accidentally querying an old index with a new model.

## Batch Indexing

Batch inputs when the provider supports it:

```ts
const result = await caller.embeddings({
  input: [
    'Password reset instructions.',
    'Invoice download instructions.',
    'Workspace invite instructions.'
  ],
  model: EMBEDDING_MODEL
});

for (const item of result.embeddings) {
  console.log(item.index, item.embedding.length);
}
```

Track usage for batch jobs:

```ts
await caller.embeddings({
  input: documents.map(d => d.text),
  model: EMBEDDING_MODEL,
  usageCallback: ({ usage }) => {
    console.log(usage.tokens.input.total, usage.costs.total);
  }
});
```

## Query Flow

A typical semantic search flow:

```ts
const query = await caller.embeddings({
  input: 'How do I invite a teammate?',
  model: EMBEDDING_MODEL
});

const queryVector = query.embeddings[0].embedding;
const matches = await vectorStore.search({
  vector: queryVector,
  topK: 5,
  filter: { embeddingModel: EMBEDDING_MODEL }
});
```

Filtering by `embeddingModel` is important during migrations or when one database stores multiple indexes.

## RAG Flow

Use retrieved text as `data` for a normal model call:

```ts
const passages = matches.map(match => match.text).join('\n\n---\n\n');

const answer = await chatCaller.call({
  text: 'Answer the user using only the provided support passages.',
  data: passages,
  endingMessage: 'If the passages do not contain the answer, say so.'
});

console.log(answer[0].content);
```

Embedding and generation do not need to use the same model. Pin the embedding model for vector consistency, then choose a chat model or preset for answer generation.

## Using Presets

Presets are useful for one-off embedding jobs:

```ts
const caller = new LLMCaller(['openai', 'gemini'], 'cheap');

const result = await caller.embeddings({
  input: 'Classify this short support message.'
});

console.log(result.metadata?.provider);
console.log(result.metadata?.model);
```

For durable retrieval indexes, prefer an exact embedding model so future queries use the same vector space.

## Custom Dimensions

Some embedding models support custom dimensions:

```ts
const result = await caller.embeddings({
  input: 'Short support message.',
  model: 'text-embedding-3-small',
  dimensions: 512
});
```

If you use custom dimensions, store the dimension count and keep it consistent for all vectors in the same index.

## Migration Strategy

When changing embedding models:

1. Create a new index or partition.
2. Re-embed documents with the new model.
3. Query the new index with the same new model.
4. Keep old and new indexes separate until migration is complete.
5. Delete the old index only after traffic has moved.

Do not compare vectors generated by different models as part of one ranking pass.
